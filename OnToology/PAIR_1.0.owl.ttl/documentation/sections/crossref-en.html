Ideac back to ToC or Class ToC

IRI: http://www.virtual-assembly.org/ontologies/1.0/pair#Idea

La notion d'Idée dans PAIR est en relation avec la notion de concept dans l'ontologie SKOS. Dans PAIR Idée est ainsi une sous-classe de SKOS:Concept URI : http://www.virtual-assembly.org/ontologies/1.0/pair#idea Définition : Dans l'ontologie PAIR, la notion d'idée se réfère à ce qui est abstrait, idéel, notionnel, conceptuel, à ce qui est de l'ordre de l'idéation, de la proposition, du virtuel (au sens philosophique). Elle se définit notamment par opposition à la notion de projet : Une idée n'est pas un projet dans la mesure où, par définition, une idée n'est pas incarnée. Une idée peut en revanche s'incarner dans un projet, ou alors, devenir un projet. Il est possible qu'un même "objet" soit perçu comme une idée ou un projet. Par exemple, les "Chemins de la transition" peuvent être perçus comme une idée, mais aussi comme un projet. C'est au regard de son intention, que le créateur des "Chemins de la transition" optera pour la catégorie idée ou projet : S'il considère qu'il s'agit juste d'un concept, il optera pour la catégorie "Idée" ; Si son intention est d'incarner cette idée dans le cadre d'un projet, il optera donc pour la catégorie correspondante. Dans l'ontologie PAIR, les idées se subdivisent en 2 catégories : - Les thèmes rassemblent les notions, les concepts, les centres d'intérêts, les "tags". Par exemple, "développement durable", "open source", "mécanique quantique", "peer to peer", pastafarisme, etc. - Les thèses rassemblent les propositions ("Et si ...", "Mettre en place un revenu universel inconditionnel"), les arguments ainsi que les problèmes et les questions (Par exemple : "Pourquoi y a t'il quelque chose plutôt que rien ?") Dans l'ontologie PAIR, thèmes et thèses constituent les 2 sous classes de de la classe Idée.

URI : http://www.virtual-assembly.org/ontologies/1.0/pair#NaturalResources Définition (wikipedia) : De manière générale, une ressource naturelle est une substance, un organisme ou un objet présent dans la nature et qui fait, dans la plupart des cas, l'objet d'une utilisation pour satisfaire les besoins (énergies, alimentation, agrément, etc.) des humains, animaux ou végétaux. La notion de Projet dans PAIR est la même que celle de Projet dans FOAF. URI : xmlns.com/foaf/0.1/project Définition : Un projet suppose une action, et donc l'incarnation d'une idée, d'une intention dans le réel. Dans l'ontologie PAIR, les Projets se subdivisent en 2 grandes catégories : - D'un côté, les initiatives, missions, activités, soit les toutes les aventures au long cours. - D'un autre côté, les événements, s'inscrivant dans une temporalité courte, avec une date de début et une date de fin. La notion d'événement réfère ici à la notion de manifestation. Elle ne recouvre pas la notion d'événement au sens d'actualité (par exemple : un tremblement de terre, un attentat, une découverte).

Social codec back to ToC or Class ToC

IRI: http://www.virtual-assembly.org/ontologies/1.0/pair#SocialCode

L’enjeu du "#CodeSocial" est de poser de manière claire et transparente le quoi, le pourquoi et le comment d’une organisation, d’un projet, d’une collaboration … Il permet aux différentes parties prenantes d’une aventure d’en co-construire une représentation commune à travers une méthode de description holistique. Le “#CodeSocial” présente la raison d’être, l’histoire, ainsi que les différents principes et modèles qui fondent nos actions : sur les plans social et humain, économique, écologique, scientifique et technologique, juridique, artistique et culturel. En rendant explicite ce qui souvent est implicite il permet à des communautés d’acteurs de travailler ensemble de manière saine, transparente et efficiente.

past project - A project this person has previously worked on. Domain: having this property implies being a Person Range: every value of this property is a Thing After a Person is no longer involved with a currentProject, or has been inactive for some time, a pastProject relationship can be used. This indicates that the Person was involved with the described project at one point. If the Person has stopped working on a project because it has been completed (successfully or otherwise), pastProject is applicable. In general, currentProject is used to indicate someone's current efforts (and implied interests, concerns etc.), while pastProject describes what they've previously been doing.
